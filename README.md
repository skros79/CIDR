# MikroTik RouterOS v7 Dynamic Address-List Sync (RAM Mode)

Скрипт для автоматической синхронизации IP-адресов из текстового файла на GitHub в `address-list` межсетевого экрана MikroTik. Работает полностью в оперативной памяти (ОЗУ) без записи на флеш-память роутера, исключая её износ.

## Основные возможности
*   **Режим зеркала:** Автоматически добавляет новые IP-адреса и точечно удаляет из MikroTik те, которые были убраны из файла на GitHub.
*   **Работа в ОЗУ:** Использование параметра `output=user` в RouterOS v7 позволяет обрабатывать данные без создания временных файлов на диске роутера.
*   **Умное обновление:** Существующие записи не пересоздаются заново (не сбрасывается Creation Time), обновляются только измененные данные.
*   **Защита от сбоев:** Если файл скачался пустым или недоступен из-за сбоя сети, скрипт прервет работу и не очистит текущие списки в MikroTik.
*   **Статистика в лог:** Выводит точное количество обработанных, добавленных и удаленных IP-адресов.
*   **Игнорирование пустот:** Пустые строки и случайные пробелы в файле автоматически очищаются и игнорируются.

## Требования
*   Устройство под управлением **RouterOS v7.x** (используются многомерные массивы).
*   Наличие настроенного DNS на MikroTik для успешного скачивания файла с GitHub.

## 1. Формат входного файла (`cidr.txt`)
Файл должен быть текстовым. Название каждого адрес-листа должно начинаться строго со знака двоеточия (`:`). IP-адреса (или подсети) указываются под заголовком списка, каждый с новой строки. Допускаются пустые строки для визуального разделения.

Пример содержимого файла:
```text
:BLOCK_SPAM
192.168.10.25
195.45.12.0/24

:VPN_ROUTING
1.1.1.1
8.8.8.8

10.0.0.5
```

## 2. Установка и настройка в MikroTik

1. На GitHub откройте ваш файл `cidr.txt`, нажмите кнопку **Raw** и скопируйте полученную прямую ссылку.
2. В интерфейсе MikroTik перейдите в **System -> Scripts** и создайте новый скрипт (например, `sync-cidr`).
3. Вставьте код скрипта в поле *Source*.
4. В самом начале кода в переменной `:local url "..."` замените адрес на вашу скопированную **Raw**-ссылку.
5. Нажмите **Apply**.

## 3. Автоматизация (Scheduler)
Для регулярного автоматического обновления списков (например, каждые 6 часов) выполните команду в терминале MikroTik или добавьте задачу в **System -> Scheduler**:

```routeros
/system scheduler add name="Sync_CIDR_Github" interval=6h on-event="sync-cidr" start-time=00:00:00
```

## ⚠️ Важное системное ограничение (Лимит 64 КБ)

В языке скриптов MikroTik RouterOS существует жесткое ограничение на максимальный размер текстовой переменной в оперативной памяти — **65 535 байт (64 КБ)**.

* **Что это значит:** Команда `/tool fetch` с параметром `output=user` загружает текст файла напрямую в переменную. Если размер вашего файла `cidr.txt` на GitHub превысит 64 КБ, RouterOS **молча обрежет** всё, что идет после этого лимита.
* **Чем это грозит:** Так как скрипт не увидит «хвост» файла из-за обрезки, он решит, что этих IP-адресов больше нет на сервере, и в режиме зеркалирования **удалит их** из адрес-листов роутера.
* **Вместимость:** 64 КБ — это примерно **3500–4000 строк** (включая пустые строки и названия списков). Если ваш файл весит меньше (например, до 30-40 КБ), скрипт работает на 100% безопасно.
